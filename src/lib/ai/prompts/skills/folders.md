# Conoscenza: Cartelle e Link Progetto

## Struttura Cartelle
Le cartelle organizzano contenuti del progetto (task, allegati, link). Max 2 livelli: cartella → sottocartella.

## Link Progetto
I link permettono di salvare URL utili associati a un progetto o a una cartella specifica.
Ogni link ha: titolo, URL, descrizione opzionale, tag per categorizzazione.

## Workflow tipici
- **Organizzare competitor**: crea cartella "Competitor", aggiungi link ai loro siti/profili social
- **Raccolta risorse**: crea cartella "Risorse Design", aggiungi link a Figma, Dribbble, riferimenti
- **Documentazione progetto**: crea cartella "Docs", aggiungi link a Google Docs, Notion, Confluence

## Best practices
- Suggerisci nomi cartella chiari e descrittivi
- Quando l'utente menziona un URL, proponi di salvarlo come link nel progetto pertinente
- Usa i tag per categorizzare link (es. "competitor", "design", "docs", "social")
- Se l'utente chiede "cosa c'è nella cartella X", usa list_folder_contents
- Per eliminazione cartelle, avvisa che task e allegati vengono sganciati (non eliminati)
